# C# / ASP.NET Core ポートフォリオ：モダンWebアプリケーション開発

このリポジトリは、C#を用いた基礎的なアプリケーション開発からスタートし、**「実際の現場で求められるモダンな技術スタックとユーザー目線のUX改善」**をテーマに独自に拡張を行ったポートフォリオです。

## 🚀 概要とアピールポイント

AI（GitHub Copilot / Gemini 等）を活用し、プロンプトエンジニアリングを駆使して一人で学習カリキュラムを完結させました。 コンソールアプリケーションによる基礎文法の習得から、Entity Framework Core を用いたデータベース連携、Razor Pages によるWebアプリケーション開発まで、17以上のチャレンジ項目をクリアしています。

また、単なるチュートリアルの消化にとどまらず、「実務でどのように課題を解決するか」を意識して以下の拡張開発を行いました。

1. **ユーザー体験(UX)の向上**:
   リスト表示だけの標準的なToDoアプリから、「タスクにはスケジュール管理が不可欠である」という仮説に基づき、外部ライブラリ(`FullCalendar`)を導入して本格的なカレンダーUIを実装しました。
2. **クラウドネイティブなインフラ構築**:
   ローカルのファイルベースDB（SQLite）から、実践的なクラウドNoSQLである **Firebase Firestore** へデータ基盤を移行しました。さらにアプリを **Docker** でコンテナ化し、**Google Cloud Run** を用いてサーバーレス環境へ本番デプロイしています。
3. **開発の背景と将来の展望（就活管理アプリへの進化）**:
   本アプリは、単なる技術学習の成果物ではなく**「自身の就職活動におけるスケジュール・タスク管理の課題を解決する」**という最終目標に向けた第一フェーズの実装です。今後はこのタスク（カレンダー）管理基盤の上に「企業ごとの選考ステータス管理機能（書類選考、面接日時の紐付け等）」を実装し、実際の就活を効率化する専用ツールへの進化を計画しています。

👉 **[本番稼働中のデモアプリはこちら（Cloud Run）](https://portfolio-todo-app-zdelu44l4q-an.a.run.app)**

## 🛠 技術スタック

- **バックエンド**: C# 13.0 / ASP.NET Core 9.0 (Razor Pages) / Google Cloud SDK
- **データベース**: Firebase Firestore (NoSQL) ※初期開発時は SQLite / Entity Framework Core を使用
- **インフラ・公開環境**: Docker / Google Cloud Run
- **フロントエンド**: Bootstrap 5.3 / FullCalendar (JavaScript)
- **開発・運用ツール**: Git / GitHub / .NET CLI

## 💻 主な機能: TodoWeb

学んだ技術を“動かす”だけでなく、“どう使えば良い体験がつくれるか”を意識して改良しています。

UX 向上への工夫
初期はシンプルな ToDo 管理アプリでしたが、「タスクにはスケジュールの見える化が欠かせない」と感じ、
JavaScript ライブラリの FullCalendar を導入して、予定管理もできるカレンダーUIを実装しました。

クラウドネイティブ化への挑戦
開発初期は SQLite を使っていましたが、運用を見据えて Firebase Firestore (NoSQL) に移行。
さらに Docker でコンテナ化し、Google Cloud Run にデプロイして本番稼働しています。

実践的なテーマ設定（就活管理アプリへ）
このアプリは単なる学習用ではなく、自分の就職活動を効率化するツールを目指しています。
今後はカレンダー機能を軸に、「企業ごとの選考ステータス管理」などを追加し、就活専用のタスク・スケジュール管理アプリへ発展させる予定です。
