# 🏆 復習テスト 3「2週間のブランクを埋める：機能拡張への挑戦」

2週間ぶりということですので、これまでに学んだ「データ・ロジック・データベース・見た目」の4要素をすべて使う復習課題を用意しました。
現在のToDoアプリに **「優先度（Priority）」** という新しい概念を追加してみましょう。

## 課題内容
既存の `TodoWeb` プロジェクトをベースに、以下の機能を追加してください。

1.  **データの拡張**: `ToDoItem` クラスに `Priority`（重要度）という項目を追加します（例：1=高、2=中、3=低）。
2.  **データベースの更新**: 新しい項目を保存できるように、マイグレーション（`migrations add` → `database update`）をやり直します。
3.  **UIの追加**: タスク追加時に、優先度を選べるようにフォーム（`<select>`など）を追加します。
4.  **表示の工夫**: 一覧画面で、優先度が高いものは「赤い文字」や「[高]」というバッジを付けて目立つようにします。

## この課題で思い出せること
- **C#クラスの定義**: データの持ち方
- **EF Coreの操作**: データベースの変更
- **Razor Pagesのロジック**: データの受け渡し
- **HTML/CSS**: 条件に応じた見た目の切り替え

今のアプリの最大の欠点は、**「アプリを止めるとデータが消えてしまう」** ことです。

## 課題内容
Entity Framework Core というツールを使って、SQLite（エキライト）という軽量なデータベースに接続します。

1.  **ライブラリのインストール**: データベースを扱うための部品を追加します。
2.  **DbContextの作成**: プログラムとデータベースを繋ぐ「窓口」を作ります。
3.  **PageModelの書き換え**: リスト（`static List`）を使っていた部分を、データベースからの読み書きに書き換えます。
4.  **確認**: アプリを一度終了してもう一度動かしても、タスクが残っていることを確認してください！

### ヒント
- **インストールするコマンド**:
`dotnet add package Microsoft.EntityFrameworkCore.Sqlite`
`dotnet add package Microsoft.EntityFrameworkCore.Design`

- **データの構造（DbContext）**:
```csharp
public class MyDbContext : DbContext
{
    public MyDbContext(DbContextOptions<MyDbContext> options) : base(options) { }
    public DbSet<ToDoItem> TodoItems { get; set; }
}
```

これまでの課題では、C#のコードの中にHTMLを「長い文字列」として書いてきました。
しかし、本番のWeb開発では **「見た目（HTML）」と「ロジック（C#）」は別のファイルに分ける** のが一般的です。

これを実現するASP.NET Coreの強力な機能が **「Razor Pages」** です。

## 課題内容
今のToDoアプリを、Razor Pagesという仕組みを使ってリファクタリング（中身の整理）してみましょう。

1.  **プロジェクト設定の変更**: `Program.cs` で Razor Pages を使うための設定を追加します。
2.  **フォルダの作成**: `Pages` という名前のフォルダを作ります。
3.  **HTMLファイルの作成**: `Pages/Index.cshtml` を作り、そこにHTMLとCSSを移動させます。
4.  **動作確認**: 見た目は変わらなくても、コードが劇的にスッキリしたことを確認してください。

### ヒント
- **Program.cs の変更点**:
```csharp
var builder = WebApplication.CreateBuilder(args);
builder.Services.AddRazorPages(); // Razor Pagesのサービスを追加
var app = builder.Build();

app.MapRazorPages(); // Razor Pagesへの道を作る
app.Run();
```
- **Index.cshtml の中身**:
`@page` というおまじないを一番上に書くことで、そのファイルがWebページになります。
