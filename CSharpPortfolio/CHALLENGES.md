# C# 入門課題：Challenge 6「リストとクラスの融合：Webアプリの基礎データ」

Webアプリでは、1つの画面に「複数のToDoオブジェクト」が表示されます。
先ほどの復習テストでは「文字のリスト」でしたが、今回は **「自作クラス（ToDo）のリスト」** を作ってみましょう。

## 課題内容
ユーザーが入力した情報を「ToDoオブジェクト」として作り、それを「リスト」に溜めていくプログラムを作成してください。

1.  **クラス**: 前回の `ToDo` クラス（NameとIsDoneを持つ）をそのまま使います。
2.  **リストの宣言**: `List<string>` ではなく、`List<ToDo>` を作ります。
3.  **入力と追加**: 
    - `while` 文の中でタスク名を受け取ります。
    - `new ToDo()` でオブジェクトを作り、Nameにその名前を入れます。（IsDoneは最初は false でOK）
    - リストにそのオブジェクトを追加します。
4.  **表示**: 最後に `foreach` で、全てのタスク名と完了状態を表示します。

### 期待される出力例
```text
タスク名を入力：
> 買い物
> 掃除
> exit

--- 登録一覧 ---
・買い物 [未完了]
・掃除 [未完了]
```

## ヒント
- **リストの宣言**: `List<ToDo> myTasks = new List<ToDo>();`
- **追加のコード**:
```csharp
ToDo item = new ToDo();
item.Name = input;
item.IsDone = false;
myTasks.Add(item);
```
- **表示のコード**:
```csharp
string status = t.IsDone ? "完了" : "未完了";
Console.WriteLine($"・{t.Name} [{status}]");
```

## 進め方
1.  `Program.cs` を書き換えて挑戦してください。
2.  エラーが出たら、メッセージをよく読んで修正してみましょう。

# C# 入門課題：Challenge 7「データの更新：タスクを完了にする」

Webアプリでは、チェックボックスをクリックしてタスクを「完了」に更新する機能が必要です。
今回は、リストの中から特定のタスクを選んで、その `IsDone` を `true` に書き換える処理に挑戦しましょう。

## 課題内容
登録したタスクの中から1つ選び、その状態を「完了」に変更して再表示してください。

1.  **事前準備**: 3つほどタスクをリストに追加しておきます（コード内で直接 `Add` してもOK）。
2.  **更新前の表示**: 一度リストを全て表示します。この時、各タスクの横に **番号（添字）** を表示してください。
3.  **選択と更新**:
    - `Console.ReadLine()` で「何番のタスクを完了にしますか？」と聞き、番号を入力してもらいます。
    - 入力された番号を使って、リストのその要素の `IsDone` を `true` に書き換えます。
4.  **更新後の表示**: 再びリストを全て表示し、正しく「完了」に変わっているか確認します。

### 期待される出力例
```text
--- 現在のタスク ---
[0] 買い物 (未完了)
[1] 掃除 (未完了)

完了にする番号を入力してください：
> 0

--- 更新後のタスク ---
[0] 買い物 (完了)
[1] 掃除 (未完了)
```

## ヒント
- **番号付きで表示する（for文）**: 
```csharp
for (int i = 0; i < task.Count; i++) {
    Console.WriteLine($"[{i}] {task[i].Name}");
}
```
- **番号で指定して更新する**:
```csharp
int index = int.Parse(Console.ReadLine());
task[index].IsDone = true;
```

## 進め方
1.  `Program.cs` を書き換えて挑戦してください。
2.  リストの「0番目」や「1番目」を正しく操作できれば成功です！
